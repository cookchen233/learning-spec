# Decomposition Rules

如何把长期学习目标拆成 `level / zone / capability`。

这里的拆解优先依据是能力结构本身，而不是主观期限。

## 1. 先定长期现实，再定能力结构

推荐顺序：

1. 写 `project-contract`
2. 拆 `level`
3. 在每个 `level` 下拆 `zone`
4. 在每个 `zone` 下拆 `capability`
5. 当前 capability 被激活时，再在 capability 目录下生成近程 `task`
6. 学习时再即时生成本次 session plan

说明：

- 可以有 18 个月、12 个月或 24 个月这样的个人期限
- 但 `level / zone / capability` 不应按月份平均切开
- 它们应按现实能力结构与阶段跃迁来拆

## 2. 先切 capability，再想近程 task 与本次 session plan

不要先写未来几个月的动作清单。

先问：

- 我到底想获得哪项能力
- 这项能力属于哪个领域
- 这些领域凑起来是否支撑某个阶段跃迁

## 3. capability 的默认切法

`capability` 应优先按真实场景能力切，而不是按知识点切。

优先切：

- 机场值机能力
- 餐厅点餐能力
- 自我介绍能力
- 看房询问能力
- 问路与听懂方向能力

不要优先切：

- 过去时
- 餐厅单词
- 听力技巧
- 一节课内容

## 4. zone 的作用

`zone` 用来组织一组同类 `capability`，并形成中等奖励。

例如：

- transport zone
- food zone
- housing zone
- social zone

判断一个 `zone` 是否合理，要看：

- 里面的能力是否明显同类
- 完成后是否真的有“拿下一块地盘”的感觉

## 5. level 的作用

`level` 不应只是能力总和，而应是阶段跃迁。

例如：

- 从无法开口，进入可完成基础生存沟通
- 从生存沟通，进入可完成日常社交沟通
- 从日常社交，进入可完成独立生活沟通

## 6. 不提前规划远期 task

正式结构只提前规划到 `capability`。

当某个 capability 进入执行期时，再围绕它生成近程 `task` 与本次 session plan。

每次学习前，可围绕当前 `capability` 生成：

- 本次 session plan
- 本次要听什么
- 本次要说什么
- 本次要录什么

一个 `capability` 通常不是靠“先想明白”获得的，而是靠下面这类即时动作逐步搭出来：

- 生成最小例子
- 对例子做多元素替换
- 跟读或模仿
- 无稿复现
- 录音回听
- 被轻微扰动后继续完成

因此：

- 人不必提前独立规划远期 task
- 系统或 AI 可以围绕当前 capability 辅助生成本次即时任务
- task 的价值不是长期规划，而是把 capability 从定义推到成立

## 7. 禁止的拆法

### 1. 把知识点当 capability

表现：

- capability 写成“掌握一般过去时”
- capability 写成“学会 50 个单词”

### 2. 把一次表现当 capability

表现：

- 只完成一次模拟，就算能力成立
- 只有一次成功，没有稳定复现

### 3. 把 zone 写成学术分类

表现：

- 结构上有分类，但没有获得感
- 完成后不觉得自己“拿下了一块领域”

### 4. 把 level 写成能力堆叠清单

表现：

- 完成若干零散 capability，但没有明显阶段跃迁

### 5. 提前铺开大量远期 task

表现：

- 为未来数周写很多细动作
- 执行时很快失真

### 6. capability 验收过弱

表现：

- 只靠一次表现就判定能力成立
- 例子数量太少
- 例子之间只替换一个词，无法证明能力稳定迁移

## 8. 最终目标

这套拆解规则最终追求的不是写出漂亮学习计划，而是：

- 让长期学习有清晰阶段结构
- 让日常推进有真实奖励
- 让即时执行保持灵活
- 让能力成立可以被证据证明
