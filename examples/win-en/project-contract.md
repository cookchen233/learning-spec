## Goal Definition

- 最终目标不是考试成绩，而是在国外生活与基础工作 / 学习中具备稳定的英语沟通能力。
- 成功标准不是“学过很多内容”，而是在高频现实场景中能独立完成基本交流、应对常见追问、修复误解并推进事项。
- 当前阶段优先建立真实生活中的听说能力，同时保留邮件、短信、表格、通知、客服聊天等必要读写能力。

## Time Reality

- 18 个月只是个人目标期限，不是正式结构的拆分依据。
- 每日稳定学习时间上限为 90 分钟。
- 正式结构不提前展开远期即时动作；每次学习前只围绕当前 capability 临时生成本次 session plan。
- 计划必须允许状态波动，不能依赖高意志力连续冲刺。

## Core Structure

- `level` 是阶段跃迁单元。
- `zone` 是奖励化领域单元。
- `capability` 是最小可验证能力单元。
- 单次学习窗口的即时动作只在临近执行时生成。

## Learning Invariants

- 学习结构优先按真实生活场景能力切分，不按语法章节、教材进度或词汇表切分。
- level / zone / capability 的划分优先服从英语能力的物理结构，而不是服从 18 个月的时间均分。
- 每个 capability 必须对应一个可重复调用的现实沟通能力，而不是一次练习成功。
- 每个 zone 必须能让人明确感觉“拿下了一块地盘”。
- 每个 level 必须意味着整体生活沟通身份发生了变化，而不是只完成了若干零散训练。
- 优先建立主成功路径能力，再补增强能力、变体能力、保护能力与纠错能力。

## Evidence Rules

- capability 成立默认依赖录音、复述、模拟对话、听后重构、真实交流记录等证据。
- 每个 capability 的验收至少包含 5 个可复现例子。
- 每个例子至少包含 3 个可替换元素，例如人物、时间、地点、物品、数量、目的、条件。
- 不接受只替换单个词的机械变化作为能力成立依据。
- 不接受“我感觉差不多会了”“我看懂了”“我背过了”作为正式证据。

## Priority Rules

- 优先顺序是：高频生存沟通 > 澄清与误解修复 > 独立生活沟通 > 行政 / 医疗 / 安全沟通 > 日常社交沟通 > 功能性工作 / 学习沟通。
- 优先拿下能立刻改变现实生活自主性的能力。
- 先建立可用，再追求自然、流利、漂亮。

## Anti-Fake-Progress Rules

- 不把“接触过这个主题”当成能力成立。
- 不把“一次表现成功”当成稳定掌握。
- 不把“学了很多内容”当成阶段进展。
- 不把抽象知识点完成当成 capability 完成。
- 不把被脚本牵引的背诵流畅当成可迁移能力。

## Feedback Rules

- 任何 capability 若长期无法成立，优先怀疑 capability 过大、证据过弱或例子变化不够。
- 任何 zone 若结构清晰但没有获得感，优先怀疑 zone 划分不对。
- 任何 level 若完成后没有明显身份跃迁感，优先怀疑 level 边界不对。
- 任何方法论问题都应记录到 `execution-state.md` 的 `Practice Feedback`。
